# 07. Embodied Interface

## Goal

Seraph should feel like a living system with legible state, not a generic chat wrapper.

## Design Direction

The current village and avatar direction is correct because it creates:

- ambient legibility
- emotional distinctiveness
- visible action state
- a stronger sense of continuity than a sterile chat UI

## Important Constraint

Embodiment only matters if it reflects real state. Cosmetic game skinning without meaningful coupling to guardian behavior is a dead end.

## Research Questions

- how should the world reflect attention, progress, stress, and momentum?
- what becomes ambient versus explicit UI?
- how should the avatar and world express trust, urgency, and confidence?
- what does the life-OS shell look like beyond the current village?
