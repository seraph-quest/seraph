# 05. Execution Plane

## Goal

Seraph needs a real action layer so guardian reasoning can translate into real-world leverage.

## Current Direction

The execution plane should combine:

- native tools for local capability
- MCP for external systems
- visible execution so actions are legible
- workflow composition on top of tool primitives

## Product Standard

Tool count is not the goal. The goal is being able to do high-value work reliably:

- inspect and update relevant local state
- browse and search effectively
- reach external systems through controlled integrations
- compose multiple steps into useful outcomes

## Research Questions

- which workflows deserve first-class support instead of freeform tool chaining?
- how should Seraph represent reusable action recipes?
- what execution patterns are most valuable for personal-guardian use cases?
