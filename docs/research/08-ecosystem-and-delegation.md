# 08. Ecosystem And Delegation

## Goal

Seraph should compound through reusable capabilities without collapsing into a messy framework product.

## Capability Layers

- skills for reusable instructions
- MCP for external tool surfaces
- specialists for focused roles
- delegation for decomposing work
- workflow composition for repeatable outcomes

## Product Constraint

Extensibility should strengthen the guardian product, not dilute it. The ecosystem must stay subordinate to the user outcome.

## Research Questions

- what abstractions should users actually author?
- when should Seraph delegate versus stay monolithic?
- how should extension quality and safety be surfaced?
- what is the minimum extension model that still compounds capability materially?
